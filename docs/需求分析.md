# 1. 人工需求分析
------
## 1.1 需求背景分析
当前公寓与社区运营存在以下问题：
### （1）房源出租效率低
- 房源依赖人工中介或线下渠道
- 信息分散，曝光不足，需要一个线上房源整合平台
- 空置周期长，收益不稳定
------
### （2）社区服务分散
- 保洁、维修等服务依赖临时渠道（微信群/熟人）
- 缺乏统一调度与标准化价格
- 响应效率低，用户体验不稳定
------
### （3）资产与物资管理低效
- 公寓家具采购缺乏统一管理与议价能力
- 二手家具与闲置物品无法有效流转
- 成本不可控
------
### （4）核心机会
系统可依托“公寓”作为供给核心，构建：
>  “房源 + 服务 + 资产管理”的本地化运营平台
------
## 1.2 产品定位
本产品是一个：
> **以公寓为核心节点的本地化房源分销 + 社区服务 + 资产管理平台**
核心目标：
- 提高房源出租效率
- 提供标准化社区服务（保洁/维修）
- 优化公寓物资采购与资产流转
------
# 2. MVP 最小可行功能点
------
##  MVP目标（必须明确）
> 在 1–3 个公寓内跑通：
- 房源成交闭环
- 服务下单闭环
- 基础资产管理记录
------
## 2.1 核心模块（必须开发）
------
###  2.1.1 房源管理与分销系统（核心变现）
#### 功能：
- 房源录入（店长端）
  - 房型
  - 价格
  - 图片
  - 状态（空置/已租/数量）
- 房源展示（用户端）
  - 浏览房源
  - 联系/预约
- 房源分销机制
  - 分销人员绑定（中介/兼职/用户）
  - 带看记录
  - 成交归属
- 佣金管理
  - 成交金额记录
  - 佣金比例配置
  - 自动结算记录（先记录，不强制支付系统）
------
###  2.1.2 社区服务系统（保洁/维修）
#### 功能：
- 服务类型：
  - 保洁
  - 基础维修
- 用户下单：
  - 地址
  - 时间
  - 描述
- 派单机制：
  - 店长指派
- 服务状态流转：
  - 待接单 → 已接单 → 服务中 → 已完成
- 评价系统：
  - 用户评价服务者
------
###  2.1.3 店长管理后台（核心控制节点）
- 房源管理
- 服务派单
- 服务人员管理
- 成交记录查看
- 佣金记录查看
###  2.1.4 采购记录模块（B端辅助）
- 供应商添加商品
- 商品展示列表
- 用户下单
- 供应商接单
###  2.1.5 二手物品信息模块（非交易系统）
发布二手家具/物品
图片 + 描述 + 联系方式
不做支付/物流

------
# 3. 用户操作流程
------
## 3.1 房源租赁流程（核心闭环）
1. 用户浏览房源
2. 预约看房
3. 分销人员带看
4. 成交
5. 佣金记录生成
------
## 3.2 社区服务流程
1. 用户提交保洁/维修需求
2. 店长或系统派单
3. 服务者接单
4. 完成服务
5. 用户评价
------
## 3.3 采购流程
1. 供应商上架产品
2. 用户根据商品列表下单
3. 供应商接单，交易完成
4. 用户确认订单完成并评价
------
## 3.4 二手发布流程
1. 店长或用户发布物品
2. 展示信息
3. 用户联系成交（线下）
------
