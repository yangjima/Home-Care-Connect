## 前置要求

- **Docker** 20.10+
- **Docker Compose** V2+
- **2GB+ 可用内存**（推荐 4GB+）

验证版本：

```bash
docker --version
docker compose version
```

## 快速启动

### 1. 进入部署目录

```bash
cd infrastructure/docker
```

### 2. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env，填入真实值（尤其是 DASHSCOPE_API_KEY）
nano .env
```

> **重要**：`DASHSCOPE_API_KEY` 是阿里云百炼 API Key，AI 服务必须此项才能正常工作。
> 获取地址：https://dashscope.console.aliyun.com/

### 3. 启动所有服务

```bash
# 方式 A：直接使用 docker compose（推荐）
docker compose up -d --build

# 方式 B：使用 PowerShell 脚本（支持分范围启动）
powershell -ExecutionPolicy Bypass -File start.ps1 -Scope All -Action Up

# 方式 C：使用 Bash 脚本
chmod +x start.sh
./start.sh
```

### 4. 等待服务就绪

```bash
# 查看服务状态
docker compose ps

# 查看所有服务健康状态（推荐等待 2-3 分钟）
docker compose ps --format "table {{.Name}}\t{{.Status}}\t{{.Ports}}"
```

等待所有服务状态变为 `healthy` 或 `running`（基础设施服务需优先就绪）。

### 5. 验证部署

| 服务 | 地址 | 验证 |
|------|------|------|
| 前端首页 | http://localhost | 页面正常加载 |
| API 网关 | http://localhost:8080/actuator/health | 显示 UP |
| Nacos 控制台 | http://localhost:8848/nacos | 登录后查看服务列表 |
| MinIO 控制台 | http://localhost:9000 | 登录后查看存储桶 |

## 环境变量

在 `infrastructure/docker/.env` 中配置：

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `MYSQL_PASSWORD` | 是 | `HomeCare2026@DB` | MySQL root 密码 |
| `REDIS_PASSWORD` | 是 | `HomeCare2026@Redis` | Redis 访问密码 |
| `JWT_SECRET` | 是 | — | JWT 签名密钥（生产环境请使用随机长字符串） |
| `JWT_EXPIRATION` | 否 | `86400000` | Token 有效期（毫秒，默认 24 小时） |
| `MINIO_ROOT_USER` | 否 | `homecareadmin` | MinIO 用户名 |
| `MINIO_ROOT_PASSWORD` | 否 | `HomeCare2026@Minio` | MinIO 密码 |
| `DASHSCOPE_API_KEY` | **是** | `sk-xxx...` | 阿里云百炼 API Key（AI 服务必需） |
| `LLM_MODEL` | 否 | `Qwen/Qwen3.5-397B-A17B` | LLM 模型（可选：Qwen/Qwen3.5-397B-A17B/qwen-max/Qwen/Qwen3.5-397B-A17B/qwen-long） |

```bash
docker compose up -d gateway user-service property-service service-order-service asset-service ai-service
```

```bash
docker compose up -d frontend
```

## 健康检查

每个容器内置健康检查，Docker Compose 使用 `condition: service_healthy` 确保依赖服务就绪后再启动。

### 手动检查服务健康状态

```bash
# 查看所有容器健康状态
docker compose ps

# 检查特定服务日志（健康检查日志）
docker compose logs --tail=20 gateway

# 直接在容器内测试健康端点
docker exec -it homecare-user-service curl -f http://localhost:8081/actuator/health
docker exec -it homecare-gateway curl -f http://localhost:8080/actuator/health
docker exec -it homecare-ai-service curl -f http://localhost:8000/health
```

### Nacos 服务注册检查

访问 http://localhost:8848/nacos （默认账号/密码：`nacos`/`nacos`），在「服务管理」→「服务列表」中确认以下 6 个服务均已注册：

- `gateway`
- `user-service`
- `property-service`
- `service-order-service`
- `asset-service`
- `ai-service`

## 停止服务

```bash
# 停止所有服务（保留数据卷）
docker compose down

# 停止并删除数据卷（清理所有数据）
docker compose down -v

# 停止特定服务
docker compose stop gateway

# 重启特定服务
docker compose restart gateway
```

## 清理数据

```bash
# 删除所有容器、网络（保留数据卷）
docker compose down

# 删除所有容器、网络、数据卷（完全清理）
docker compose down -v

# 删除所有未使用的镜像
docker image prune -f

# 删除未使用的构建缓存
docker builder prune -f
```

> **警告**：`docker compose down -v` 会删除 MySQL、Redis、MinIO、Nacos 的所有持久化数据，请确认操作后果后再执行。

## 常见问题

### Q1: 微服务启动失败，显示 `Connection refused` 或 `Nacos` 相关错误

**原因**：服务启动时 Nacos/MySQL/Redis 尚未就绪。

**解决**：
```bash
# 确认基础设施服务健康
docker compose ps mysql redis nacos

# 等待基础设施就绪后重启微服务
docker compose restart gateway user-service property-service service-order-service asset-service
```

### Q2: 前端页面加载空白或 502 Bad Gateway

**原因**：Gateway 未就绪，或 `nginx.conf` 中的代理地址不正确。

**解决**：
```bash
# 确认 Gateway 健康
docker compose ps gateway
docker logs homecare-gateway

# 重启前端
docker compose restart frontend
```

### Q3: AI 服务返回 `API key not provided` 或 401 错误

**原因**：`DASHSCOPE_API_KEY` 未配置或配置错误。

**解决**：
1. 编辑 `infrastructure/docker/.env`，填入真实的 API Key
2. 重启 AI 服务：`docker compose up -d ai-service`
3. 检查日志：`docker logs homecare-ai-service`

### Q4: 健康检查一直失败（`curl: (22) Failed to connect`）

**原因**：容器内 curl 未安装，或服务启动时间超过 health check 的 `start_period`。

**解决**：
```bash
# 查看服务启动日志
docker logs homecare-user-service

# 如果是启动较慢，增加 healthcheck 的 start_period
# 编辑 docker-compose.yml 中对应服务的 healthcheck.start_period
```

### Q5: 图片上传后无法访问

**原因**：MinIO 未就绪，或 Bucket 未创建。

**解决**：
```bash
# 确认 MinIO 健康
docker compose ps minio

# 登录 MinIO 控制台 (http://localhost:9000) 检查 bucket 是否存在
# Bucket 应由应用启动时自动创建，如未创建需手动创建：
# homecare-property、homecare-product、homecare-secondhand
```

### Q6: Nacos 控制台看不到注册的服务

**原因**：微服务启动失败，或网络隔离。

**解决**：
```bash
# 查看微服务日志
docker compose logs user-service --tail=50

# 确认容器在同一个网络
docker network inspect homecare-net

# 重启微服务
docker compose restart gateway user-service property-service service-order-service asset-service
```

### Q7: MySQL 初始化 SQL 未执行

**原因**：`init.sql` 仅在 MySQL 数据卷为空时执行（首次启动）。

**解决**：
```bash
# 清理 MySQL 数据卷后重启（会重新执行 init.sql）
docker compose down -v
docker compose up -d mysql
# 等待 MySQL 完全就绪后再启动其他服务
docker compose up -d
```

## 开发工作流

### 1. 本地开发（非 Docker）

后端微服务在本地 IDE 中运行，使用 Docker 中的基础设施服务：

```bash
# 启动基础设施
docker compose up -d mysql redis minio nacos

# 在 IDE 中启动各微服务
# 各服务的 Nacos/Redis/MySQL 地址使用容器名称：
# Nacos: nacos:8848
# MySQL: mysql:3306 (root/你的 MYSQL_PASSWORD)
# Redis: redis:6379 (密码: 你的 REDIS_PASSWORD)
# MinIO: http://minio:9001
```

### 2. 前后端联调（本地前端 + Docker 后端）

```bash
# Docker 中启动所有后端服务和基础设施
docker compose up -d --build mysql redis minio nacos gateway user-service property-service service-order-service asset-service ai-service

# 前端在宿主机以开发模式运行
cd ../../frontend
npm install
npm run dev
# 前端开发服务器默认运行在 http://localhost:5173
```

### 3. 前端打包后部署

```bash
# 在 frontend 目录执行
npm install
npm run build

# Docker Compose 会自动使用 frontend/dist/ 构建镜像
docker compose build frontend
docker compose up -d frontend
```

### 4. 使用 Makefile

项目根目录提供了 Makefile：

```bash
# 启动所有服务
make up

# 停止所有服务
make down

# 重启所有服务
make restart

# 查看服务状态
make ps

# 查看指定服务日志
make logs-svc SERVICE=gateway

# 仅构建镜像
make build

# 清理未使用镜像
make clean
```

## 生产环境注意事项

1. **修改默认密码**：生产环境务必修改 `.env` 中的所有默认密码
2. **JWT_SECRET**：使用足够长的随机字符串，不使用默认值
3. **CORS 配置**：编辑 `services/gateway/src/main/resources/application.yml` 中的 `allowedOrigins`，限制为真实域名
4. **HTTPS**：在前端 Nginx 配置 SSL 证书，或使用反向代理（如 Nginx/Caddy）
5. **数据卷持久化**：确保 `docker-compose.yml` 中的数据卷映射到宿主机的可靠路径
6. **资源限制**：根据实际流量，在 `docker-compose.yml` 中为各容器添加 `deploy.resources.limits`
7. **日志收集**：建议配置 Docker 日志驱动（如 `json-file` + 日志收集服务）
8. **监控**：可接入 Prometheus + Grafana 监控各服务状态
