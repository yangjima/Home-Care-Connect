# 第二部分：AI 模块架构（LangGraph 多 Agent）

## 2.1 设计理念

**核心目标：** 让用户通过自然对话，告诉 AI 想要什么（租房子、找保洁、买东西），AI 自动理解意图并路由到对应业务模块，生成结构化结果返回给用户。

**技术选型：**

- **LangGraph** — 构建有状态的多 Agent 协作图，支持条件分支、历史记忆、工具调用
- **DashScope SDK** — 通过阿里云百炼平台调用千问模型（qwen-plus 等）
- **Redis** — 存储对话历史（LangGraph Checkpoint），支持跨会话上下文

## 2.2 Agent 架构图



```
                          ┌─────────────────────┐
                          │    User Message      │
                          │  "我想找一个月租     │
                          │   2000以下的房子"     │
                          └──────────┬──────────┘
                                     ▼
                          ┌─────────────────────┐
                          │   Router Agent      │
                          │  (意图分类路由)       │
                          │  分类: property      │
                          └──────────┬──────────┘
                                     │
           ┌─────────────────────────┼─────────────────────────┐
           │                         │                         │
           ▼                         ▼                         ▼
  ┌────────────────┐        ┌────────────────┐        ┌────────────────┐
  │ Property Agent │        │ Service Agent  │        │ Procurement   │
  │   (房源管家)    │        │   (服务管家)    │        │   Agent        │
  │ intent=property│        │ intent=service │        │  (采购管家)    │
  └───────┬────────┘        └───────┬────────┘        │ intent=procure │
          │                         │                 └───────┬────────┘
          ▼                         ▼                         ▼
  调用 Property Svc          调用 Service Order      调用 Asset Svc
  查询房源数据               Svc 查询服务类型         查询商品列表
          │                         │                         │
          └─────────────────────────┼─────────────────────────┘
                                    ▼
                          ┌─────────────────────┐
                          │   Response Agent    │
                          │  (整合结果 + 生成回复) │
                          └──────────┬──────────┘
                                     │
                          ┌──────────▼──────────┐
                          │   Save to Redis     │
                          │   (对话历史持久化)    │
                          └─────────────────────┘
```

## 2.3 各 Agent 职责详解

| Agent                 | 触发关键词                         | 能力                                                         | 调用后端                                 |
| --------------------- | ---------------------------------- | ------------------------------------------------------------ | ---------------------------------------- |
| **Router Agent**      | 无（所有消息入口）                 | 理解用户意图，分类为 `property` / `service` / `procurement` / `general` | 无                                       |
| **Property Agent**    | 租房、看房、房源、公寓、房价、租金 | 理解找房需求 → 调用 Property Service → 返回房源列表/详情     | `http://property-service/api/properties` |
| **Service Agent**     | 保洁、维修、清洁、疏通、开锁       | 理解服务需求 → 调用 Service Order Svc → 返回服务类型/价格/预约 | `http://service-order/api/services`      |
| **Procurement Agent** | 买、购物、商品、采购、家具、二手   | 理解采购需求 → 调用 Asset Svc → 返回商品列表/二手信息        | `http://asset-service/api/procurement`   |
| **Response Agent**    | 无（聚合节点）                     | 整合各 Agent 结果，调用 LLM 生成自然语言回复                 | 无                                       |
| **Fallback Agent**    | 其他所有                           | 直接调用 LLM 回答问题，不调用任何后端服务，友好引导用户或提供帮助 | 无（直接回答）          |

## 2.4 LangGraph 状态定义



```python
# AI 服务状态定义
class AgentState(TypedDict):
    """LangGraph 状态定义"""
    messages: List[BaseMessage]           # 对话历史（LangChain 格式）
    user_id: str                          # 用户ID（从JWT解析）
    intent: str                            # 当前识别到的意图：property/service/procurement/general
    context: dict                          # 业务上下文（房源条件、服务类型等）
    result: Any                            # 后端服务返回的原始数据
    response: str                           # 最终要返回给用户的文本
```

## 2.5 核心路由逻辑（伪代码）



```python
# LangGraph 流程定义
from langgraph.graph import StateGraph, END

def build_agent_graph():
    graph = StateGraph(AgentState)
    
    # 添加节点
    graph.add_node("router", router_agent_node)
    graph.add_node("property", property_agent_node)
    graph.add_node("service", service_agent_node)
    graph.add_node("procurement", procurement_agent_node)
    graph.add_node("response", response_agent_node)
    graph.add_node("fallback", fallback_agent_node)
    
    # 入口 → Router
    graph.set_entry_point("router")
    
    # Router 根据 intent 分发
    graph.add_conditional_edges(
        "router",
        route_by_intent,
        {
            "property": "property",
            "service": "service", 
            "procurement": "procurement",
            "general": "fallback",
        }
    )
    
    # 各 Agent → Response
    graph.add_edge("property", "response")
    graph.add_edge("service", "response")
    graph.add_edge("procurement", "response")
    graph.add_edge("fallback", "response")
    
    # Response → 结束
    graph.add_edge("response", END)
    
    return graph.compile()

def route_by_intent(state: AgentState) -> str:
    """Router Agent 核心逻辑：调用千问判断用户意图"""
    last_message = state["messages"][-1].content
    prompt = f"""
    分析用户消息，判断意图类型，只能返回以下四种之一：property, service, procurement, general

    判断规则：
    - 涉及租房、看房、房源、公寓、房价、租金 → property
    - 涉及保洁、维修、清洁、疏通、开锁等服务 → service
    - 涉及买、购物、商品、采购、家具、二手 → procurement
    - 其他所有问题（闲聊、百科、计算、日常问答等）→ general

    用户消息：{last_message}

    返回格式：只需返回英文单词，不要其他内容。
    """
    response = call_qwen(prompt)
    return response.strip()
```

## 2.6 对话历史持久化（Redis）



```python
# LangGraph Checkpointer 使用 Redis
from langgraph.checkpoint.redis import RedisSaver

checkpointer = RedisSaver(
    redis=get_redis_client(),
    key_prefix="ai:checkpoint:"
)

# 每个用户的对话线程通过 thread_id 隔离
# thread_id = user_id（从 JWT 解析）
config = {"configurable": {"thread_id": user_id}}
graph.invoke(user_input, config=config, checkpointer=checkpointer)
```

**Redis Key 设计：**

- `ai:checkpoint:{user_id}` — LangGraph 检查点（对话状态快照）
- `ai:history:{user_id}` — 简化版对话历史（JSON）
- `session:expire:{user_id}` — Session 过期时间（TTL 30 分钟无活动自动清理）

```
POST /api/ai/chat
Headers: Authorization: Bearer <jwt_token>
Body: { "message": "我想找一个月租2000以下的房子" }
Response: { "intent": "property", "reply": "好的，正在为您查找...", "data": {...} }

```

**前端 Vue 组件调用链：**



```
用户输入 → <AiChatWidget> 
         → AI Service (/api/ai/chat)

```

```nginx
# nginx/nginx.conf
events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    upstream gateway {
        server gateway:8080;
    }

    server {
        listen 80;
        server_name localhost;
        root /usr/share/nginx/html;
        index index.html;

        # Vue 静态资源
        location / {
            try_files $uri $uri/ /index.html;
        }

        # API 反向代理
        location /api/ {
            proxy_pass http://gateway;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            
            # WebSocket 支持（AI 流式对话）
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_read_timeout 86400;
        }

        # MinIO 静态资源（图片/视频直连）
        location /minio/ {
            proxy_pass http://minio:9000/;
        }
    }
}
```

## 6.5 开发规范

### 命名规范

| 类型        | 规范                        | 示例                                    |
| ----------- | --------------------------- | --------------------------------------- |
| Java 类名   | UpperCamelCase              | `PropertyService`, `PropertyController` |
| Java 方法名 | lowerCamelCase              | `getPropertyById`, `createOrder`        |
| Java 常量   | UPPER_SNAKE_CASE            | `ORDER_STATUS_PENDING`                  |
| 数据库表名  | snake_case，模块前缀        | `property`, `service_order`, `sys_user` |
| 数据库列名  | snake_case                  | `rent_price`, `created_at`              |
| Vue 组件名  | PascalCase                  | `PropertyList.vue`, `AiChatWidget.vue`  |
| API 路径    | RESTful，`/api/模块/资源`   | `/api/property/properties`              |
| Git 分支    | `feature/`, `fix/`, `docs/` | `feature/property-module`               |

### Git 工作流程



```
main (生产分支，受保护)
├── develop (开发分支)
│   ├── feature/property-module
│   ├── feature/ai-agent
│   ├── feature/service-order
│   ├── fix/xxx-bug
│   └── docs/api-doc
```

**Commit 规范：**



```
feat: 新增功能
fix: 修复 bug
refactor: 重构
docs: 文档更新
chore: 构建/工具/配置
test: 测试相关
style: 代码格式（不影响功能）
```

### 代码风格

- **Java**: 遵循阿里 Java 开发规范（可配合 Alibaba Cloud Java 代码规范插件）
- **Python**: 遵循 PEP 8（`black` 格式化 + `ruff` lint）
- **Vue**: 遵循 Vue 官方风格指南，Options API 或 Composition API 均可（推荐 Composition）
- **TypeScript**: 严格模式，使用 Interface 而非 Type（简单场景可用 type）

### 接口规范

- 所有接口必须携带 `code`、`message`、`data` 统一响应
- 分页查询统一参数：`page`、`size`
- 日期格式：ISO 8601（`2026-04-16T10:00:00Z`）或 `yyyy-MM-dd HH:mm:ss`
- 敏感字段（密码）不在响应中返回
