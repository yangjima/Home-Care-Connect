## Phase 0 · 基础设施

> **最重要的阶段，所有服务都依赖它，必须先跑通再往下走。**

- [ ] **P0-1 初始化项目仓库结构**
  - 创建 monorepo 目录：`services/` `frontend/` `infrastructure/`
  - 子目录：`services/gateway` `services/user-service` `services/property-service` `services/service-order` `services/asset-service` `services/ai-service`
  - 配置根目录 `.gitignore`（排除 `.env`、`target/`、`node_modules/`、`__pycache__/`）
  - 创建 `.env.example`（含 MYSQL_PASSWORD / JWT_SECRET / MINIO_ROOT_USER / MINIO_ROOT_PASSWORD / DASHSCOPE_API_KEY）
  - 标签：`5min` `git`

- [ ] **P0-2 编写 docker-compose.yml**
  - 包含服务：MySQL 8.0、Redis 7-alpine、MinIO latest、Nacos v2.2.3
  - 每个服务配置 healthcheck 和 volumes 数据持久化
  - MySQL 挂载 `./infrastructure/mysql/init.sql` 自动建表
  - 验证：`docker compose up -d` 后四个服务均 healthy
  - 标签：`30min` `docker`

- [ ] **P0-3 编写数据库初始化 SQL (init.sql)**
  - 按顺序建以下 13 张表（注意外键依赖顺序）：
    1. `store` — 门店表
    2. `sys_user` — 用户表（role ENUM: admin/store_manager/service_staff/distributor/user）
    3. `property` — 房源表（facilities JSON，status ENUM: vacant/occupied/reserved）
    4. `property_image` — 房源图片表
    5. `distributor` — 分销人员表（bind_code UNIQUE，commission_rate 默认 0.01）
    6. `property_viewing` — 看房记录表（status ENUM: pending/completed/cancelled）
    7. `commission` — 佣金记录表（status ENUM: pending/settled/cancelled）
    8. `service_type` — 服务类型表（category ENUM: cleaning/repair/other）
    9. `service_staff` — 服务人员表（skills JSON，status ENUM: available/busy/off）
    10. `service_order` — 服务订单表（status ENUM: pending/assigned/accepted/in_progress/completed/cancelled）
    11. `review` — 评价表（target_type ENUM: property/service/staff）
    12. `procurement_product` — 采购商品表
    13. `secondhand_item` — 二手物品表（condition ENUM: like_new/good/fair）
  - 标签：`45min` `mysql`

- [ ] **P0-4 配置 Nacos 共享配置文件**
  - `infrastructure/nacos/shared.yaml`：DB 连接、Redis、MinIO 连接信息
  - 各服务占位 yaml：`gateway.yaml` `user-service.yaml` `property-service.yaml` `service-order.yaml` `asset-service.yaml` `ai-service.yaml`
  - 标签：`20min` `nacos`

## Phase 1 · 后端 — User Service (:8081)

- [ ] **P1-1 搭建 Spring Boot 项目骨架**
  - `pom.xml` 引入：`spring-boot-starter-web` `spring-boot-starter-data-jpa` `spring-boot-starter-data-redis` `spring-cloud-starter-alibaba-nacos-discovery` `spring-cloud-starter-alibaba-nacos-config` `jjwt` `lombok`
  - `application.yml` 配置 Nacos 地址，引用共享配置
  - 目录结构：`controller/` `service/` `repository/` `entity/` `dto/` `common/` `util/`
  - 标签：`20min` `spring-boot`

- [ ] **P1-2 实现统一响应 + 全局异常处理**
  - `Result<T>` 类（code / message / data / timestamp）
  - `PageResult<T>` 分页响应（records / total / page / size）
  - `BusinessException` + `ErrorCode` 枚举
  - `GlobalExceptionHandler` 处理 BusinessException 和通用异常
  - 标签：`15min` `公共组件`

- [ ] **P1-3 注册 & 登录接口 (JWT)**
  - `POST /auth/register`：手机号+密码，BCrypt 加密，支持 bind_code 注册时绑定分销关系
  - `POST /auth/login`：校验密码，返回 `accessToken` + `refreshToken` + 用户信息
  - `POST /auth/refresh`：RefreshToken 换新 AccessToken
  - `GET /auth/me`：返回当前登录用户信息
  - `JwtUtil`：生成/解析 Token，从 Token 提取 userId/role
  - 标签：`45min` `JWT`

- [ ] **P1-4 用户 CRUD + 角色管理**
  - `GET /users`：分页查询（仅 admin 可访问）
  - `GET /users/{id}`：用户详情
  - `PUT /users/{id}`：更新用户信息（头像/昵称等）
  - `PUT /users/{id}/password`：修改密码（校验旧密码）
  - `GET /users/staff`：获取服务人员列表（供派单使用）
  - 标签：`30min`

- [ ] **P1-5 JWT 鉴权过滤器（供 Gateway 复用）**
  - `JwtAuthFilter`：从 Header 取 `Authorization: Bearer <token>`，解析写入 `X-User-Id` 和 `X-User-Role` 请求头
  - 将 JwtUtil 抽取到公共依赖模块，供其他服务通过 OpenFeign 传递用户上下文
  - 标签：`20min` `JWT`

## Phase 2 · 后端 — Property Service (:8082)

- [ ] **P2-1 房源 CRUD + 分页查询**
  - `POST /properties`：创建房源（仅 store_manager）
  - `GET /properties`：分页查询，支持参数：`type` `minPrice` `maxPrice` `status` `keyword` `storeId` `page` `size`
  - `GET /properties/{id}`：房源详情（含图片列表、浏览量 +1）
  - `PUT /properties/{id}`：更新房源
  - `DELETE /properties/{id}`：删除房源
  - `PATCH /properties/{id}/status`：更新状态（vacant/occupied/reserved）
  - 标签：`60min`

- [ ] **P2-2 房源图片上传 (MinIO)**
  - `POST /properties/{id}/images`：接收 MultipartFile，上传到 MinIO bucket `homecare-property`，返回访问 URL
  - `DELETE /properties/images/{imageId}`：删除图片（同步删除 MinIO 文件）
  - MinIO SDK 配置：endpoint / accessKey / secretKey 从 Nacos 读取
  - 标签：`30min` `MinIO`

- [ ] **P2-3 分销绑定码 & 分销员注册**
  - `POST /distributors/bind`：用户带 bind_code 绑定分销关系，创建 distributor 记录
  - `GET /distributors`：分销人员列表（仅管理员）
  - `GET /distributors/{id}/stats`：个人分销统计（总成交/累计佣金/待结算佣金）
  - 标签：`20min`

- [ ] **P2-4 预约看房流程**
  - `POST /viewings`：提交看房预约（property_id / user_id / distributor_id / appointment_time）
  - `GET /viewings`：看房记录列表（按角色过滤：用户看自己的，管理员看全部）
  - `PATCH /viewings/{id}/status`：更新状态（pending→completed/cancelled）
  - 标签：`30min`

- [ ] **P2-5 成交登记 & 佣金自动记录**
  - `POST /deals`：店长登记成交（property_id / distributor_id / deal_price）
  - 自动计算：`commission_amount = deal_price × commission_rate`，写入 `commission` 表（status=pending）
  - 同时更新房源状态为 occupied，更新 distributor 的 total_deals 和 total_commission
  - `GET /deals`：成交记录列表
  - 标签：`30min`

- [ ] **P2-6 佣金记录查询**
  - `GET /distributors/commissions`：当前分销员的佣金记录，支持 status 过滤和分页
  - `PATCH /commissions/{id}/settle`：店长手动标记结算（status: pending→settled）
  - 标签：`20min`

## Phase 3 · 后端 — Service Order (:8083)

- [ ] **P3-1 服务类型管理 CRUD**
  - `GET /types`：获取服务类型列表（公开）
  - `POST /types`：新增服务类型（仅 admin）
  - `PUT /types/{id}`：更新服务类型
  - `DELETE /types/{id}`：删除服务类型
  - 标签：`20min`

- [ ] **P3-2 服务人员管理**
  - `POST /staff`：添加服务人员（仅 store_manager）
  - `GET /staff`：服务人员列表（含技能标签/状态/评分）
  - `GET /staff/{id}`：服务人员详情
  - `PUT /staff/{id}/status`：更新状态（available/busy/off）
  - 标签：`25min`

- [ ] **P3-3 用户下单接口**
  - `POST /orders`：提交服务订单（service_type_id / address / appointment_time / description）
  - 自动生成 order_no（格式：`SVC + yyyyMMdd + 6位序号`），初始 status=pending
  - 标签：`20min`

- [ ] **P3-4 店长指派 & 服务人员接单**
  - `PATCH /orders/{id}/assign`：店长指派服务人员（staff_id），status: pending→assigned
  - `PATCH /orders/{id}/accept`：服务人员接单，status: assigned→accepted
  - 权限校验：assign 仅 store_manager，accept 仅被指派的 service_staff
  - 标签：`20min`

- [ ] **P3-5 订单状态流转全链路**
  - `PATCH /orders/{id}/status`：更新为 in_progress（服务开始）
  - `POST /orders/{id}/complete`：确认完成，status→completed，记录 completed_at
  - `DELETE /orders/{id}`（逻辑删除）：取消订单，status→cancelled
  - `GET /orders`：订单列表（用户看自己的，staff 看分配给自己的，manager 看全部），支持 status 过滤
  - `GET /orders/{id}`：订单详情
  - 标签：`30min`

- [ ] **P3-6 评价系统**
  - `POST /reviews`：用户评价（target_type: service/staff，rating 1-5，content，images JSON）
  - `GET /reviews`：评价列表（按 target_type + target_id 查询）
  - 限制：同一用户对同一 target 只能评价一次
  - 标签：`20min`

## Phase 5 · 后端 — API Gateway (:8080)

- [ ] **P5-1 Spring Cloud Gateway 路由配置**
  - 5 条路由规则：
    - `/api/user/**` → `lb://user-service`
    - `/api/property/**` → `lb://property-service`
    - `/api/service/**` → `lb://service-order`
    - `/api/asset/**` → `lb://asset-service`
    - `/api/ai/**` → `lb://ai-service`
  - 路由配置存放在 Nacos `gateway.yaml`，支持动态更新
  - 标签：`20min` `gateway`

- [ ] **P5-2 JWT 鉴权全局过滤器**
  - 白名单（无需鉴权）：`POST /api/user/auth/register`、`POST /api/user/auth/login`、`GET /api/property/properties`、`GET /api/service/types`
  - 其余请求：验证 Token → 解析 userId/role → 写入下游请求头 `X-User-Id` / `X-User-Role`
  - Token 无效返回 401，权限不足返回 403
  - 标签：`30min` `JWT`

- [ ] **P5-3 跨域 CORS 配置**
  - 允许来源：开发环境 `http://localhost:5173`，生产环境配置域名
  - 支持 `Authorization` 请求头
  - 支持 WebSocket Upgrade（供 AI 流式接口）
  - 标签：`10min`

## Phase 6 · AI Service (FastAPI + LangGraph)

> **建议在前端开发前完成，否则 AiChatWidget 组件无法联调。**

- [ ] **P6-1 FastAPI 项目骨架 + Nacos 注册**
  - 目录结构：`routers/` `agents/` `llm/` `tools/` `memory/` `middleware/`
  - `main.py`：创建 FastAPI app，挂载路由，启动时注册到 Nacos
  - `middleware/auth.py`：从请求头 `X-User-Id` 读取 userId（由 Gateway 注入）
  - `config.py`：从环境变量读取 DASHSCOPE_API_KEY / REDIS_HOST / 各服务 URL
  - 标签：`20min` `FastAPI`

- [ ] **P6-2 DashScope 千问调用封装**
  - `llm/qwen.py`：封装 `call_qwen(prompt: str) -> str`
  - 使用 `dashscope` SDK，model 默认 `qwen-plus`
  - 支持流式输出 `call_qwen_stream(prompt)` → 生成器
  - 统一错误处理（网络超时、API 限流重试）
  - 标签：`20min` `DashScope`

- [ ] **P6-3 AgentState 定义 + LangGraph 图构建**
  - `agents/state.py`：
    ```python
    class AgentState(TypedDict):
        messages: List[BaseMessage]
        user_id: str
        intent: str   # property/service/procurement/general
        context: dict
        result: Any
        response: str
    ```
  - `agents/graph.py`：构建 StateGraph，添加5个节点（router/property/service/procurement/response/fallback），配置条件边
  - 标签：`30min` `LangGraph`

- [ ] **P6-4 Router Agent — 意图分类**
  - `agents/router.py`：调用千问，Prompt 要求只返回4种意图之一
  - 关键词映射：租房/房源/公寓 → property；保洁/维修/疏通 → service；买/采购/商品/二手 → procurement；其他 → general
  - 返回值写入 `state["intent"]`
  - 标签：`20min` `Agent`

- [ ] **P6-5 Property / Service / Procurement Agent**
  - `tools/property_tools.py`：调用 `http://property-service:8082/properties`，传入筛选参数
  - `tools/service_tools.py`：调用 `http://service-order:8083/types` 获取服务列表
  - `tools/asset_tools.py`：调用 `http://asset-service:8084/products` 和 `/secondhand`
  - 各 Agent 将后端返回数据写入 `state["result"]` 和 `state["context"]`
  - 标签：`45min` `Agent`

- [ ] **P6-6 Response Agent + Fallback Agent**
  - `agents/response_agent.py`：取 `state["result"]`，调用千问生成自然语言回复，写入 `state["response"]`
  - `agents/fallback_agent.py`：直接调用千问回答通用问题，友好引导用户
  - Response 结构：`{ intent, reply, data, redirect }`（redirect 为前端跳转路径）
  - 标签：`25min` `Agent`

- [ ] **P6-7 Redis 对话历史持久化**
  - `memory/redis_checkpointer.py`：使用 `langgraph.checkpoint.redis.RedisSaver`
  - Key 规则：`ai:checkpoint:{user_id}`，TTL 30 分钟（无活动自动清理）
  - `POST /api/ai/chat`：每次调用时传入 `config={"configurable": {"thread_id": user_id}}`
  - `GET /api/ai/history`：返回对话历史
  - `DELETE /api/ai/history`：清空历史
  - 标签：`20min` `Redis`

- [ ] **P6-8 WebSocket 流式对话接口**
  - `routers/websocket.py`：`WS /api/ai/ws?token=<jwt_token>`
  - 消息帧协议：
    - 客户端发：`{ "type": "message", "content": "..." }`
    - 服务端推：`intent` → 多个 `chunk` → `result` → `done`
  - 每个 chunk 对应千问流式输出的一个 token
  - 标签：`30min` `WebSocket`

## Phase 7 · 前端 Vue 3

- [ ] **P7-1 Vue 3 项目搭建 + 路由 + Pinia**
  - 技术栈：Vite + Vue 3.4 + Element Plus + Axios + Pinia + Vue Router 4
  - 路由守卫：未登录重定向 `/login`，已登录不可再访问 `/login`
  - Axios 拦截器：请求加 `Authorization: Bearer <token>`，响应统一处理 401（清空 Token 跳登录）
  - Pinia：`store/user.js`（userId/role/token）、`store/ai.js`（对话状态）
  - 标签：`20min` `vue3`

- [ ] **P7-2 登录 / 注册页面**
  - 对应原型：`01-login.html`
  - Tab 切换：账户登录 / 快速注册
  - 登录：POST `/api/user/auth/login` → 写入 Pinia + localStorage
  - 注册：POST `/api/user/auth/register`，支持分销码填写
  - 第三方登录按钮（UI 占位即可）
  - 标签：`30min`

- [ ] **P7-3 首页 + 通用 AppHeader 组件**
  - 对应原型：`01-home.html`
  - AppHeader：Logo + 导航菜单 + 登录/用户头像，登录后显示头像和姓名
  - 首页内容：Banner 轮播（静态占位）、快速入口4格、热门房源3列、推荐服务4格
  - 标签：`45min`

- [ ] **P7-4 房源列表 + 详情页**
  - 对应原型：`02-property-list.html` `03-property-detail.html`
  - 列表：筛选栏（类型/价格区间/状态）+ 卡片网格，分页
  - 详情：图片轮播（Element Plus Carousel）、房源信息、设施标签、预约看房按钮（弹出日期选择器）
  - 标签：`60min`

- [ ] **P7-5 服务列表 + 预约下单页**
  - 对应原型：`04-service-list.html` `05-service-booking.html`
  - 列表：服务类型卡片（图标/名称/起步价），点击进入预约页
  - 预约表单：地址输入、日期时间选择、问题描述，提交调 `POST /api/service/orders`
  - 标签：`45min`

- [ ] **P7-6 采购商城 + 二手市场页**
  - 对应原型：采购 `01-purchase.html`，二手 `02-secondhand.html`
  - 采购：分类 Tab + 商品卡片（图片/名称/价格/库存），下单弹窗
  - 二手：分类 Tab + 瀑布流卡片（成新度/价格/卖家/位置），发布按钮打开发布表单
  - 标签：`45min`

- [ ] **P7-7 个人中心 + 我的订单**
  - 对应原型：`03-profile.html` `04-my-orders.html`
  - 个人中心：用户信息卡片、快捷入口（我的订单/看房/佣金/收藏）、常用功能菜单列表
  - 我的订单：Tab（全部/待支付/进行中/已完成/已取消）+ 订单卡片（状态/金额/操作按钮）
  - 标签：`40min`

- [ ] **P7-8 分销员佣金页**
  - 对应原型：`05-my-commissions.html`
  - 统计卡片：累计佣金 / 待结算 / 已结算 / 成交单数
  - 分销绑定码：展示 bind_code，复制链接按钮
  - 佣金记录表格：订单号/房源/成交金额/佣金比例/佣金金额/状态/时间，状态筛选下拉
  - 标签：`30min`

- [ ] **P7-9 管理后台 Dashboard + 房源/服务管理**
  - 对应原型：`01-dashboard.html` `02-property-mgmt.html` `03-order-mgmt.html` `04-service-mgmt.html` `05-staff-mgmt.html`
  - Dashboard：数据看板（总订单数/本月成交/待结算佣金/活跃服务人员）
  - 房源管理：表格列表 + 新增/编辑/删除/状态切换
  - 服务订单管理：订单列表 + 指派服务人员操作（下拉选 staff）
