# 系统后台前后端联调清单

本文档用于“系统后台”上线前的联调验收，所有数据均走真实接口和数据库。

## 1. 环境检查

- 网关可访问：`/api/*` 路由正常转发。
- `service-order-service` 与 `asset-service` 启动日志无建表/补字段报错。
- 管理员账号可登录，JWT 头可透传 `X-User-Id`、`X-User-Role`。

## 2. 系统后台基础可用性

- 进入 `/admin/dashboard` 成功，左侧菜单显示正常。
- 菜单高亮随路由变化，刷新后保持正确。
- 非后台角色访问 `/admin/*` 被拦截。

## 3. 添加服务联调

页面：`/admin/services/new`

- 提交字段：`name/category/price/unit/description/icon`。
- 接口：`POST /api/service/service-types`。
- 验证点：
  - 平台管理员创建后 `status=1`（已上架）。
  - 商家创建后 `status=2`（待审核）。
  - 在 `/admin/services` 或审批页可查询到新增记录。

## 4. 添加商品联调

页面：`/admin/products/new`

### 本地商城商品

- 接口：`POST /api/asset/procurement-products`。
- 核对落库：`procurement_product.name/category/price/stock/status/supplier_id`。

### 跳蚤市场商品

- 接口：`POST /api/asset/secondhand-items`。
- 核对落库：`secondhand_item.title/category/price/condition/status/publisher_id`。

## 5. 商品管理联调

页面：`/admin/products`

- 查询：
  - `GET /api/asset/procurement-products`
  - `GET /api/asset/secondhand-items`
- 编辑：
  - `PUT /api/asset/procurement-products/{id}`
  - `PUT /api/asset/secondhand-items/{id}`
- 审核：
  - `POST /api/asset/procurement-products/{id}/approve-listing`
  - `POST /api/asset/procurement-products/{id}/reject-listing`
  - `POST /api/asset/secondhand-items/{id}/approve-listing`
  - `POST /api/asset/secondhand-items/{id}/reject-listing`
  - `POST /api/asset/secondhand-items/{id}/submit-listing`
- 删除：
  - `DELETE /api/asset/procurement-products/{id}`
  - `DELETE /api/asset/secondhand-items/{id}`

## 6. 烟囱测试（最小可用链路）

1. 管理员登录 -> 进入系统后台 -> 菜单可导航。
2. 新增服务 -> 服务管理可见 -> 审批页状态一致。
3. 新增本地商城商品 -> 商品管理可见 -> 审批通过后前台可见。
4. 新增跳蚤市场商品 -> 商品管理可见 -> 驳回后可“重新提交”。
5. 删除商品后列表实时更新，数据库记录同步变化。

## 7. 失败场景检查

- 参数不完整时，前端表单有明确提示。
- 接口失败时，页面不崩溃且有错误提示。
- 无权限操作时，后端返回 403，前端给出可理解反馈。
